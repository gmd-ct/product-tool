# Product Planning Command

Create a comprehensive product plan based on the given requirements.

## Instructions
1. Analyze the current codebase and existing features
2. Understand the user's requirements: $ARGUMENTS
3. Break down the work into phases
4. Identify dependencies and risks
5. Output a structured plan

## Output Format
```markdown
# Product Plan: [Title]

## Objective
[Clear statement of what we're building]

## Phases
### Phase 1: [Name]
- [ ] Task 1
- [ ] Task 2

### Phase 2: [Name]
- [ ] Task 1
- [ ] Task 2

## Dependencies
- [List dependencies]

## Risks & Mitigations
- Risk: [description] -> Mitigation: [approach]

## Success Criteria
- [Measurable criteria]
```
