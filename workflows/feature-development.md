# Feature Development Workflow

## Overview
This workflow guides the end-to-end process of developing a new feature.

## Stages

### 1. Planning
**Agent**: Product Agent
**Tasks**:
- [ ] Gather requirements
- [ ] Write user stories
- [ ] Define acceptance criteria
- [ ] Prioritize in backlog

**Output**: Feature specification document

---

### 2. Design
**Agent**: Dev Agent
**Tasks**:
- [ ] Technical design
- [ ] API design (if applicable)
- [ ] Database schema changes (if applicable)
- [ ] Component architecture

**Output**: Technical specification

---

### 3. Implementation
**Agent**: Dev Agent
**Tasks**:
- [ ] Set up feature branch
- [ ] Implement core functionality
- [ ] Write unit tests
- [ ] Handle edge cases

**Output**: Working code with tests

---

### 4. Testing
**Agent**: QA Agent
**Tasks**:
- [ ] Code review
- [ ] Integration testing
- [ ] Performance testing
- [ ] Security review

**Output**: Test report

---

### 5. Documentation
**Agent**: Docs Agent
**Tasks**:
- [ ] Update API docs
- [ ] Update user guide
- [ ] Update changelog

**Output**: Updated documentation

---

### 6. Deployment
**Tasks**:
- [ ] Merge to main
- [ ] Deploy to staging
- [ ] Smoke tests
- [ ] Deploy to production

**Output**: Feature live in production

## Completion Criteria
- [ ] All tests passing
- [ ] Documentation updated
- [ ] Feature deployed and verified
- [ ] Stakeholders notified
