# ChatPRD - Users, Personas & Use Cases

## Target Users

### Primary Personas

#### 1. Solo Product Managers
**Role**: Only PM at a startup or small company

**Daily Activities**:
- Write PRDs for new features
- Create user stories for engineering
- Document requirements quickly
- Communicate with stakeholders

**Why They Choose ChatPRD**:
- No team to review work with
- Need to move fast
- Can't spend hours on documentation
- AI acts as PM partner

**Quote**: "I'm the only PM at a company of 9 people, so if I can 10x myself, it's going to mean we can go even longer without hiring another of me."

#### 2. Startup Founders
**Role**: Building product without dedicated PM

**Daily Activities**:
- Define product requirements
- Communicate vision to engineers
- Make product decisions
- Document features

**Why They Choose ChatPRD**:
- No PM expertise
- Need professional documentation
- Time-constrained
- Building product themselves

#### 3. Product Managers at Scale-ups
**Role**: PM at growing company with increasing documentation needs

**Daily Activities**:
- Write multiple PRDs per week
- Align cross-functional teams
- Document complex features
- Coach junior team members

**Why They Choose ChatPRD**:
- Volume of documentation high
- Need consistency
- Value coaching features
- Integration with Linear/Notion

#### 4. Product Leaders
**Role**: Director/VP/CPO overseeing product teams

**Daily Activities**:
- Review team PRDs
- Provide feedback
- Set documentation standards
- Coach product managers

**Why They Choose ChatPRD**:
- Scale review capacity
- Ensure quality standards
- Template management
- Team workspace features

#### 5. Technical Leads/Engineers
**Role**: Engineering lead needing clear requirements

**Daily Activities**:
- Understand product requirements
- Break down work into tasks
- Clarify specifications
- Create technical docs

**Why They Choose ChatPRD**:
- Get clearer requirements
- Linear integration for workflow
- Technical spec generation
- Direct mention in issues

#### 6. Neurodivergent Professionals
**Role**: PMs with ADHD or other neurodivergent conditions

**Why They Choose ChatPRD**:
> "Basically made a coach that not only makes the really tough deep work tasks easier to start and finish, but also gives me a sense of being supported as a neurodivergent person."

- Helps with executive function
- Reduces blank page paralysis
- Provides structure
- Supportive AI partner

## Use Case Examples

### Use Case 1: Rapid PRD Generation from Idea

**Scenario**: PM has a feature idea and needs to document it quickly

**Workflow**:
1. Input simple idea: "Add dark mode to our app"
2. Answer ChatPRD's clarifying questions
3. Receive comprehensive PRD draft
4. Review and customize sections
5. Export to Notion/Linear
6. Share with engineering team

**Value**: "I was super burnt out last week but had a really cool side project idea... ChatPRD came up with an INCREDIBLE scaffold for me. Was one of the first moments of real UX joy I've experienced in quite some time."

**Time**: From hours to ~30 minutes

### Use Case 2: Improving Existing PRD

**Scenario**: PM wrote a PRD but wants feedback before sharing

**Workflow**:
1. Paste half-written PRD into ChatPRD
2. Request feedback and improvements
3. AI identifies gaps and weak points
4. Receive suggestions to strengthen narrative
5. Iterate on document
6. Export polished version

**Value**: "I just pasted a half-written PRD and it gave me some awesome feedback to strengthen the narrative."

### Use Case 3: Fractional PM Client Work

**Scenario**: Fractional PM needs to ramp up quickly with new client

**Workflow**:
1. Input client's product context
2. Generate PRD templates customized for client
3. Quickly document initial requirements
4. Use ChatPRD to hit ground running
5. Answer remaining questions with AI

**Value**: "Started working with a potential new fractional client. Showed them chatprd and were able to hit the ground running. Made it super easy to answer remaining questions and figure out tech requirements."

### Use Case 4: Linear Issue Enhancement

**Scenario**: Engineering needs clearer issue descriptions

**Workflow**:
1. Engineer mentions @chatprd in Linear issue
2. Requests improvement to issue description
3. ChatPRD rewrites with PM best practices
4. Generates sub-issues with done criteria
5. Continues conversation in thread for refinement

**Value**: Clear requirements without PM bottleneck

### Use Case 5: Idea to Prototype in 20 Minutes

**Scenario**: PM wants to visualize feature before full development

**Workflow**:
1. Generate PRD with ChatPRD
2. Export PRD to v0 or bolt.new
3. Generate working UI prototype
4. Share prototype with stakeholders
5. Gather feedback before engineering commitment

**Value**: Validate ideas with minimal investment
